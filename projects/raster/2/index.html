Introduction

In your first project Brush, you learned how to edit arrays of pixels to create your own digital images. Now, you're going to explore the area of image processing to manipulate existing images. In Lab 03: Convolution, you have already started to learn about convolution. In this assignment, you will be implementing a few common image processing algorithms using convolution to create filters like those you might see in image editing applications such as Photoshop or even the photos app on your phone.

Edge Detection/Sobel

Implement an edge detection filter, which highlights areas with drastically different intensities, or high frequency. Edge detection can be done by convolving using Sobel kernels on a grayscale image that should be normalized to the [0, 1] range first. 3.1 Loading Images

You can open an image using the “Load Image” button in the Filter menu, which will import the image and display it on the canvas. Once you have loaded in an image the image data becomes the canvas data. Accessing the canvas data as you have done in previous assignments is then equivalent to indexing into the image's pixels, and you can modify the pixels by modifying the RGBA structs as before. As a reminder, you will need to call Canvas2D's displayImage function to update the canvas when you make changes to the RGBA data vector.

3.2 filterImage()

In Canvas2D, there is a function called filterImage() which you will need to fill in. When the Filter button is pressed in the GUI, this function will be called, which should then apply the selected filter to the image loaded into the canvas. As you did in Brush, you can figure out which option has been selected in the GUI using the Settings class.

You should not be doing all of your convolution in the filterImage() function. It is up to you in designing your program to decide where to organize the majority of the filter calculations.

3.3 Design

Think about how to avoid redundancy and inefficiency in your code. Some of the tasks such as blurring and edge detection which have a constant kernel use the same computations for doing convolution, which can be factored out into its own function. Recall also that many of these 2D kernels are separable, and that you can use the horizontal and vertical kernels to do 2 separate convolutions in succession that use less operations. Even scaling, which does not use a fixed kernel, can be done in the x and y direction as two independent transformations.
