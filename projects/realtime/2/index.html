Project 6: Logan Give Us Another Name

You can find the algo for this project here.

Introduction

In the last project, you set up the foundation to render a static scene using a scene parser, a camera, and VAOs+ tesselation data.

In this project, you will extend your code to handle camera movement, finish your phong illumination model, and learn to apply post-processing effects to your scene using FBOs.

Deliverables

  • Phong Lighting
    • Point
    • Spot Lights
    • Light Attenuation
  • Camera Movement
    • Implement translation on wasd (left, right, backward, forward) and ctrl (down) and space (up) presses
    • Track key state and update continuously
    • Implement rotation on mouse move
      • Horizontal mouse movement = rotate about up vector
      • Vertical mouse movement = rotate about u vector (refer to slides)
      • Implement look vector axis-angle rotation
  • FBOs
    • 2 Effects, at least one from each of the following lists
      • Per-pixel: invert, grayscale, hue shift
      • Kernel-based: sharpen, box blur, edge detection
  • Texture Mapping
    • Adding UV coodinates to your tesselator
    • Passing Texture data into the Shader
    • Using Texture Data within the shader to set the output color

Responsive Camera Movement

In the last project, you created a static camera object meaning you could only view the single from a single point of view. For this part of the assignment, you will need to add several methods for transforming the camera. This will allow you to interactively move the camera around the scene!

Specifications

Your camera should move according to these specifications:

  • W/S: camera translation forward and backward respectivly along the look vector.
  • A/D: camera translation left and right respectivly in the x-z plane.
  • Space/Ctrl: camera translation up and down respectivly along the up vector (0,1,0)
  • Mouse movement: camera rotation

For mouse movement specifically.

Moving the mouse horizontally should rotate the look vector around the up vector to make sure the camera does not roll about its w axis.

Moving the mouse vertically should move the look vector in the plane made by the look and up vector.

You are going to need to figure out how to track the mouse position over time as the QT events do not explicitly track movement, only current position.

Additionally you will need to perform some sensitivity conversion from pixel moved and button presses to translation units. We leave this to you as the ideal sensitivity is ultimately down to personal preference.

Restrictions

You may NOT use the following GLM functions, we expect you to have a good enough understanding of transform matricies to construct helper functions for each of these: glm::lookAt() glm::scale() glm::rotate() glm::translate()

FBOs.

Using what you learned within labs 10 and 11, you are going to implement a pair of realtime post-processing filter effects using Frame Buffer Objects.

Filter Options

You are required to implement at least one per-pixel filter and one kernel based filter.

Per-pixel: - Color Invert - Gray Scale - Hue shift

Kernel-based: - sharpen - box blur - edge detection (sobel)

Realtime Texture Mapping

In addition to the phong illumation model extensions we have asked you to implement, the final feature we ask you to implement is realtime texture mapping for your 4 basic primitives.

This will require you to go back and modify your tesselation algorithm to generate UV coordinates for each vertex. Additionally you will need to add this attribute to your VAO and vertex shader to get the necessary information to the fragment shader where you can actually integrate the texture color.

Think of this as a test as to the extensiblity of your design and the chance to show off how easy it is to add new features to your implementation!
