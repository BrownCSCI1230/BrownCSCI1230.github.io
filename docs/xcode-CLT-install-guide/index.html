CSCI1230 Xcode Command Line Tool Install Guide

To install QtCreator, you must first have the Xcode Command Line Tools installed locally.

Apple provides a complete development enviroment named Xcode for programmers. If you are developing applications for macOS, iOS, watchOS and tvOS, you must install the full Xcode application.

However, if you are not developing for an Apple device, normally you do not necessarily need to install the full Xcode application which is about 20GB for a clean install but it can go up to 40GB!

The light-weight component is called Xcode Command Line Tools and we provide four ways to install it.

Option 1: Install the full Xcode application (Recommended)

Method 1: Using App Store

If you have time and don't want to bother, you can simply download the Xcode application from App Store. This is the easiest approach but it can take hours depending on your device and network connection.

Method 2: Manual Install

Another approach is to download and unpack the Xcode package manually from this website. It will take a significantly less amount of time comparing to directly installing from App Store.

  1. Go to the website and log in with your AppleID.
  2. Select the latest non-beta Xcode release. You can refer what the latest version is from App Store.

  1. Download the xip package and unpack it locally. There will be a Xcode application after you unpack the package. Move the Xcode package to the Applications folder.

After you have the full Xcode application installed, open the Xcode once, there should be a prompt look like below asking to install Xcode Command Line Tools (if you have never installed Xcode Command Line Tools previously). You may also be asked to enter the password for your device to verify the install.

To verify the install, use command

xcode-select -p

and you should see the following output.

/Applications/Xcode.app/Contents/Developer

Option 2: Install the Xcode Command Line Tools via terminal

Normally, this would be the recommended approach, because it takes so much less time and disk space. However, the Xcode Command Line Tools installed using this approach is located at a different location, and the Qt installer seems to have issues detecting these locations.

During the installation of Qt, the installer may pop up alert for Xcode Command Line Tools not installed. You can try and see if you can dismiss the alert and continue the install process. Some TAs have verified on a brandly new macOS environment that this alert is not a blocking one and will disappear after you dismiss it. The assignments can also be built without any issue.

However, we still cannot guarantee that this will not cause any issue since the amount of samples we have is too small. We recommend that you do not go with this approach unless you are really struggling with disk space.

Method 1: Directly install the Xcode Command Line Tools in Terminal

Launch the terminal on your device and type the following command.

xcode-select --install

You will see the following prompt, click Install and the install process will begin.

When completed, you can also verify the install using command xcode-select -p. The output should be

/Library/Developer/CommandLineTools
